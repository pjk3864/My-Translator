404 나오는 이유: GitHub 저장소 "맨 위"에 index.html이 없어서 그래요.
Vercel은 저장소 "맨 위(루트)"에 있는 파일만 배포해요. 그래서 반드시 아래처럼 올려야 해요.

========================================
★ 올바른 올리기 (index.html이 맨 위에 보여야 함)
========================================

1. GitHub에서 새 저장소 만들기 (또는 기존 저장소 열기)

2. "Add file" → "Upload files" 클릭

3. ★ 중요 ★
   - "번역기" 폴더 "안"으로 들어가세요.
   - 폴더 안에 보이는 것만 골라서 올리세요:
     * index.html (파일)
     * api (폴더 통째로)
     * README.md
     * vercel.json
     * package.json
     * 배포_404_해결.md (있으면)
   
   즉, "번역기" 폴더 자체를 끌어다 놓지 말고,
   "번역기" 폴더를 연 다음 → 그 안의 내용만 전부 선택해서 올리세요.

4. 그러면 GitHub 저장소 첫 페이지에 "index.html"이 바로 보여야 해요.
   (폴더 하나만 보이고 그 안을 클릭해야 index.html이 보이면 → 잘못된 거예요.)

5. Commit changes 클릭

6. Vercel에서 해당 프로젝트 → Deployments → 맨 위 ⋯ → Redeploy

========================================
지금 저장소가 "폴더 하나만" 보인다면
========================================

방법 A: 새로 올리기
- 새 저장소 만들고, 위 3번처럼 "번역기 안의 내용만" 다시 올리기

방법 B: Vercel에서 Root Directory 지정
- Vercel → 프로젝트 → Settings
- 왼쪽에서 "Build and Deployment" (또는 "Build and development") 클릭
- 아래로 내려서 "Root Directory" 찾기
- 거기에 지금 GitHub에서 보이는 그 폴더 이름 입력 (예: 번역기)
- Save 후 Deployments → Redeploy
