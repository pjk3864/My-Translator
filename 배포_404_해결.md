# 404 NOT_FOUND 나올 때 체크리스트

Vercel에서 **404: NOT_FOUND** 가 나오면, 대부분 **프로젝트 루트에 index.html이 없다**고 인식해서 그렇습니다. 아래만 순서대로 확인하세요.

---

## 1. GitHub 저장소 구조 확인

1. 브라우저에서 **GitHub 저장소** 열기 (예: `https://github.com/내아이디/저장소이름`).
2. **맨 위에 보이는 파일 목록**을 확인합니다.

**케이스 A:** `index.html` 이 **바로 보일 때** (api 폴더, README 등과 같은 단계)
- → **2단계**에서 **Root Directory 는 비우기**.

**케이스 B:** `index.html` 이 **안 보이고**, **폴더 하나**만 보일 때 (예: `번역기` 폴더)
- 그 **폴더 이름**을 적어 둡니다 (예: `번역기`).
- → **2단계**에서 Root Directory에 **그 폴더 이름**을 넣습니다.

---

## 2. Vercel에서 Root Directory 설정

1. [vercel.com](https://vercel.com) 로그인 → **해당 프로젝트** 클릭.
2. 위 메뉴에서 **Settings** 클릭.
3. 왼쪽 메뉴에서 **Build and Deployment** (또는 **Build and development**) 선택.
4. **Root Directory** 항목 찾기. (General이 아니라 **빌드/배포** 설정 쪽에 있음)
   - **케이스 A** (index.html이 저장소 맨 위에 있을 때):  
     - **비워 두기** (또는 `.` 한 칸만 있어도 됨).  
     - **Edit** → 칸 비우고 **Save**.
   - **케이스 B** (파일들이 폴더 안에 있을 때):  
     - **Edit** 클릭 → **번역기** (또는 1단계에서 적어 둔 폴더 이름) 입력 → **Save**.
5. 저장 후 **Deployments** 탭으로 이동.
6. 맨 위 배포 오른쪽 **⋯** (점 세 개) → **Redeploy** 클릭.
7. 1~2분 기다린 뒤, 배포된 **주소**로 다시 접속해 보기.

---

## 3. 그래도 404면

- GitHub에 **index.html**이 **정말 있는지** 다시 확인 (위 1단계).
- Vercel **Deployments** 탭에서 맨 위 배포가 **Ready** (초록) 인지 확인.
- 배포된 **URL**이 `https://프로젝트이름.vercel.app` 형태인지 확인하고, **그 주소**로 접속했는지 확인.

Root Directory를 **파일이 있는 위치**에 맞추는 것이 404 해결의 핵심입니다.
